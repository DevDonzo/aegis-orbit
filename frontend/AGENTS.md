# Unified Frontend Skill: `frontend-design-unified`

This project uses one consolidated frontend skill profile that combines guidance from:

1. `anthropics/skills` → `frontend-design`
2. `design/frontend-design` from `codex-skills-registry`
3. `finfin/awesome-frontend-skills` as a pattern/reference library

## Design Intent

Build a mission-grade interface that feels modern, high-trust, and visually elegant while remaining fast and accessible.

## Rules for this Frontend

1. **Visual quality first**: cinematic gradients, depth, and clear hierarchy without clutter.
2. **Readable by default**: high contrast, generous spacing, typographic rhythm.
3. **Systems feel**: every card, control, and result should feel part of one coherent design language.
4. **Motion with purpose**: subtle transitions only where they improve comprehension.
5. **Accessibility baseline**: keyboard reachable controls, visible focus states, semantic structure.
6. **Operational clarity**: prioritize confidence signals (risk band, probability, status, API errors).
7. **Production-minded UX**: no dead controls, useful defaults, clear loading and error states.

## Applied Scope

These rules apply to everything under `frontend/`.
